# Сессия: 24.06.2026 — Root cause fix: LLM-driven диалог

## Файл с планом

`FIX-ROOT-CAUSE.md` — полный план изменений.

## 🔴 Коренная причина 1: Сериализация сессий

**Файл:** `src/db/sessions.py:83`

`json.dumps(state, default=str)` превращал `HumanMessage`/`AIMessage` в строки.
При загрузке `isinstance(m, HumanMessage)` → `False`, `msg.content` → `AttributeError`.

**Фикс:** `messages_to_dict` / `messages_from_dict` из `langchain_core.messages`.

## 🔴 Коренная причина 2: Риджидный pipeline вместо LLM-driven диалога

### Старая архитектура (удалена)

```
START → route_from_start()
  ├─ book, handle_tour_selection (по current_step)
  └─ classify → route_by_request_type()
       ├─ tour_search → clarify (3 итерации → форсированный search_tours)
       ├─ faq → faq_search → escalate|END
       ├─ complaint/talk_to_manager → escalate
       ├─ booking → book
       └─ greeting/unknown → clarify → ... (тот же pipeline)
```

**Проблема:** LLM не управлял диалогом. После 3 итераций `clarify` принудительно форсировал `search_tours`, даже если пользователь не про туры.

### Новая архитектура

```
START → converse() (LLM с полным контекстом)
  ├─ action="respond" → END
  ├─ action="search" → search_tours → present_tours → END|escalate
  ├─ action="book" → book → END
  ├─ action="escalate" → escalate → END
  └─ action="faq" → faq_search → END
```

## Баги, найденные по ходу

### 1. `_build_filters` падает с int budget

**Лог:** `AttributeError: 'int' object has no attribute 'replace'` в `google_sheets.py:213`.

**Причина:** LLM вернул `"budget": 1000` (int, не str). `_build_filters` вызывал `1000.replace(...)`.

**Фикс:** В `converse()` нормализация `budget` → `str`, `travelers` → `int`. + защита в `_build_filters`.

### 2. Сообщения без mid игнорируются (pre-existing)

**Файл:** `src/main.py:237`

**Причина:** `continue` в else-ветке при отсутствии `mid` полностью пропускал обработку.

**Фикс:** Убран `continue` — теперь обрабатываются без дедупликации.

### 3. LLM слишком eager в search

**Симптом:** на сообщение "давайте второе" LLM шёл в `search` вместо `respond`.

**Фикс:** Усилен `CONVERSE_PROMPT` — явные примеры когда использовать `respond`, правило "лучше respond, чем false search".

### 4. `raise_for_status` AsyncMock warning (pre-existing)

**Файл:** `tests/test_instagram.py:210`

**Причина:** `mock_response.raise_for_status = AsyncMock()` — синхронный метод мокался как async.

**Фикс:** `AsyncMock()` → `Mock()`.
