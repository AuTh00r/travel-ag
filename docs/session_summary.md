# Описание сессии отладки токенов Meta Graph API

## 1. Суть проблемы и текущий статус
Пользователь и ассистент пытаются настроить интеграцию бота для Instagram-аккаунта `_shelter_0`. 
* **Проблема:** Обращения к Meta Graph API (например, к эндпоинту `/me`) с предоставленным access token возвращают ошибку `Invalid OAuth access token - Cannot parse access token`, в то время как веб-инструмент отладки (Meta Debug Tool) показывает статус токена как **«Действительно» (Valid: true)**.
* **Важное разделение сущностей:** Активная подписка на вебхуки (зеленый статус) означает лишь готовность Meta отправлять POST-запросы. Она **не равна** рабочему маркеру доступа. Без валидного токена окончательная привязка аккаунта к приложению не происходит.

## 3. Выявленное противоречие (Парадокс)
* **Поведение в Debugger:** Токен полностью валиден, имеет тип `User`, обладает всеми необходимыми `scopes` для работы с Instagram Business API.
* **Поведение в cURL / API-запросах:** При отправке прямых запросов Meta возвращает ошибку парсинга (`Cannot parse access token`). Второй токен (`IGAAOhqQ2...ZDZD`) выдает аналогичную ошибку.

---

## 4. Рабочие гипотезы и следующие шаги

1. **Проблема кодировки или скрытых символов (Наиболее вероятная):** При копировании строки токена из интерфейса мессенджера или блокнота могли продублироваться или исказиться невидимые символы (пробелы, переносы строк, экранирование спецсимволов).
   * *Действие:* Сравнить побайтово строку, отправляемую в cURL, и строку, вставляемую в Debugger. Скопировать токен повторно непосредственно из Graph API Explorer.
2. **Сбой контекста вызова API:**
   Запрос отправляется не к тому узлу (например, попытка вызвать Graph API без указания правильной версии или базового URL, из-за чего Meta не может распарсить заголовок авторизации).
   * *Действие:* Проверить отправку через главную функцию бота (`send_message`), чтобы исключить ошибки ручного формирования cURL-запроса.
