# Запуск бота на собственном сервере — полный гайд с нуля

Этот гайд написан для человека, который **не знаком** с серверами, Docker и сетями. Делай по шагам сверху вниз — ничего пропускать не нужно.

---

## 0. Что вообще происходит и зачем (прочитай, это важно)

Наш бот — это программа, которая:
1. постоянно работает и слушает интернет на «адресе» (порт `8000`);
2. когда клиент пишет в Instagram, Meta (владелец Instagram) **присылает наш серверу сообщение** на специальный адрес — это называется **webhook**;
3. бот думает (через DeepSeek), отвечает клиенту и шлёт уведомления в Telegram.

Главная проблема домашнего/офисного компьютера: Meta умеет стучаться **только по защищённому публичному адресу вида `https://что-то.com/webhook/instagram`**. А твой компьютер дома спрятан за роутером и не виден из интернета. Поэтому половина гайда — про то, как дать боту такой публичный HTTPS-адрес.

Будет два способа сделать адрес:
- **Способ А — Cloudflare Tunnel (рекомендую новичку).** Ничего на роутере настраивать не надо, HTTPS бесплатный и автоматический. Туннель сам «пробивает» путь из интернета к твоему компьютеру.
- **Способ Б — DuckDNS + проброс портов на роутере.** Классика. Сложнее, требует доступ к роутеру и «белый» IP от провайдера.

Если не уверен — делай **Способ А**.

## 1. Какой нужен сервер

«Сервер» — это просто компьютер, который:
- **включён 24/7** (если выключишь — бот не отвечает, сообщения в это время теряются);
- имеет **минимум 2 ГБ свободной оперативной памяти** (бот грузит в память модель для поиска по FAQ);
- подключён к интернету по кабелю (стабильнее, чем Wi-Fi).

**Операционная система.** Гайд написан под **Ubuntu Linux 22.04+** (самый частый выбор для серверов, всё бесплатно). Если у тебя Windows — смотри короткий раздел [«Если сервер на Windows»](#если-сервер-на-windows) в конце, основная логика та же.

Команды ниже выполняются в **терминале** (на Ubuntu: приложение «Terminal»). Строки, начинающиеся с `sudo`, спросят пароль — это нормально.

## 2. Ставим Docker (один раз)

**Docker** — это «коробка», которая запускает наш бот со всеми зависимостями одинаково на любом компьютере. Нам не придётся вручную ставить Python, библиотеки и т.д.

Скопируй и выполни по очереди:

```bash
# обновить список программ
sudo apt update

# поставить вспомогательные пакеты
sudo apt install -y ca-certificates curl git

# добавить официальный репозиторий Docker
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] \
  https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# поставить сам Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# разрешить запускать docker без sudo (нужно перелогиниться после этой строки)
sudo usermod -aG docker $USER
```

После последней команды **выйди из системы и зайди снова** (или перезагрузи компьютер), чтобы право заработало.

Проверка, что Docker жив:

```bash
docker run hello-world
```

Если увидел «Hello from Docker!» — всё ок.

## 4. Настраиваем секреты: `.env` и `credentials.json`

Бот не запустится без ключей доступа. Их два файла.

### 4.1. Файл `.env`

В проекте есть шаблон `.env.example`. Сделай из него рабочий `.env`:

```bash
cp .env.example .env
nano .env
```

Откроется текстовый редактор `nano`. Заполни **реальными** значениями (их даёт тот, кто настраивал Instagram/DeepSeek/Telegram — скорее всего у тебя они уже есть с прошлого запуска):

| Переменная | Что это | Где взять |
|---|---|---|
| `DEEPSEEK_API_KEY` | Ключ к «мозгу» бота | Личный кабинет DeepSeek |
| `DEEPSEEK_MODEL` | Модель | Оставь как в текущем рабочем `.env` |
| `INSTAGRAM_APP_SECRET` | Секрет приложения Meta | Meta for Developers → твоё приложение |
| `INSTAGRAM_ACCESS_TOKEN` | Токен доступа к Instagram | Meta for Developers |
| `INSTAGRAM_VERIFY_TOKEN` | Любое придуманное тобой слово-пароль | Придумай сам, запомни — пригодится в шаге 7 |
| `INSTAGRAM_PAGE_ID`, `INSTAGRAM_IG_USER_ID`, `INSTAGRAM_APP_ID` | ID страницы/аккаунта/приложения | Meta for Developers |
| `GOOGLE_TOURS_SHEET_ID`, `GOOGLE_REQUESTS_SHEET_ID` | ID Google-таблиц | Из URL таблицы |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота уведомлений | От @BotFather |
| `TELEGRAM_MANAGER_CHAT_ID` | Куда слать уведомления | ID чата менеджера |
| `DUCKDNS_TOKEN`, `DUCKDNS_DOMAIN` | Нужны только для Способа Б | duckdns.org |

> Если переноси́шь бота со старого сервера — проще всего скопировать туда уже готовый `.env` целиком, а не заполнять заново.

Сохранить в `nano`: `Ctrl+O`, затем `Enter`, затем выйти `Ctrl+X`.

### 4.2. Файл `credentials.json` (Google Sheets)

Это файл-ключ для записи заявок в Google-таблицы. Положи его **в корень папки проекта** (`~/travel-agent-bot/credentials.json`). Если переносишь со старого сервера — скопируй оттуда тот же файл.

Проверь, что оба файла на месте:

```bash
ls -la .env credentials.json
```

Должны быть видны обе строки. Если `credentials.json` нет — заявки в Google Sheets писаться не будут (но бот в целом запустится).

## 5. Первый запуск (локально, без интернета снаружи)

Запускаем бот в Docker:

```bash
docker compose up -d --build
```

- `--build` — собрать «коробку» (в первый раз идёт 5–10 минут, качается PyTorch и т.д. — это нормально);
- `-d` — запустить в фоне, чтобы бот работал, даже когда закроешь терминал.

Проверим, что бот живой (на самом компьютере):

```bash
curl http://localhost:8000/health
```

Ответ `{"status":"ok"}` = бот работает. 🎉

Посмотреть логи (что бот делает):

```bash
docker compose logs -f
```

Выйти из просмотра логов — `Ctrl+C` (бот при этом продолжит работать).

> На этом этапе бот работает, но **из интернета он ещё не доступен** — Instagram до него не достучится. Это чиним в шаге 6.

## 6. Даём боту публичный HTTPS-адрес

Выбери ОДИН способ.

> **Если у фирмы уже есть домен с сайтом — покупать новый не нужно.** Бот вешается на отдельный **поддомен** (`bot.домен-фирмы.com`), сайт остаётся на основном адресе и **не ломается** — это разные адреса. Подробности и что спросить у программиста фирмы — в разделе [«Если домен уже есть»](#если-домен-уже-есть-поддомен-для-бота) ниже.

### Если домен уже есть (поддомен для бота)

Сайт фирмы живёт на `домен.com` / `www.домен.com`, а бот мы повесим на **поддомен** `bot.домен.com`. Это не мешает сайту — адреса независимые. Покупать домен не надо (экономия — единственная плата отпадает).

Какой путь подойдёт — зависит от того, **где сейчас управляется DNS домена**. Это знает программист фирмы — задай ему два вопроса:
1. Где управляется DNS — у регистратора, на хостинге или уже в Cloudflare?
2. Можно ли завести поддомен `bot.домен.com` под бота?

**Путь 1 — перевести домен в Cloudflare (бесплатно, рекомендуется).**
Программист меняет у домена **nameservers на Cloudflare**. При добавлении домена Cloudflare **автоматически копирует все существующие DNS-записи** (сайт, почта и т.д.), поэтому сайт продолжает работать там же — меняется только «кто отвечает на DNS-запросы», а не хостинг сайта. Дальше — обычный **Способ А** ниже, поддомен `bot.домен.com`.
⚠️ Перенос nameservers — единственный чувствительный шаг, он касается живого домена. **Делать должен программист фирмы** (или с его согласия), чтобы не зацепить сайт/почту. Шаг обратимый и стандартный, но координация обязательна.

> Если домен **уже в Cloudflare** — вообще идеально: ничего переносить не надо, программист просто подтверждает, и сразу делаешь Способ А.

**Путь 2 — не трогать nameservers.**
Если домен трогать рискованно, программист добавляет **одну DNS-запись**: поддомен `bot.домен.com` → **публичный IP вашего сервера**. А на сервере используешь **Способ Б** (Caddy сам выдаст HTTPS).
Минус: нужен «белый» IP у сервера и проброс портов 80/443 — у домашнего интернета этого часто нет.

После того как поддомен заведён, дальше иди по выбранному способу (А или Б), подставляя везде `bot.домен.com` вместо примеров с `ТВОЙ-ДОМЕН`.

### Способ А — Cloudflare Tunnel (рекомендуется) 🟢

Не требует белого IP и настройки роутера. Туннель сам «пробивает» путь из интернета к твоему компьютеру, HTTPS-сертификат бесплатный и автоматический.

#### Сколько это стоит

| Компонент | Цена |
|---|---|
| Аккаунт Cloudflare | **0 ₽** (карта не нужна для базового туннеля) |
| Cloudflare Tunnel + HTTPS + DNS | **0 ₽**, без лимита трафика |
| **Доменное имя** | **единственная плата — ~1–12 $/год** |

Помесячной платы нет. Единственный расход — доменное имя (нужно, чтобы у бота был постоянный адрес; бесплатные временные адреса не годятся — Instagram требует стабильный webhook-URL).

> ⚠️ У многих регистраторов 1-й год дешёвый по акции, а продление дороже. Смотри **цену продления**, а не только первого года.

#### Шаг 0: купить домен на Spaceship и привязать к Cloudflare

> Если домен у фирмы **уже есть** — этот шаг пропусти, см. раздел [«Если домен уже есть»](#если-домен-уже-есть-поддомен-для-бота) выше: домен не покупаешь, а заводишь поддомен `bot.домен.com`.

Подойдёт любой регистратор, ниже — на примере **Spaceship** (дёшево, бесплатная WHOIS-приватность, от компании Namecheap).

1. Зарегистрируйся на https://spaceship.com и купи домен, например `mytravelbot.xyz` (дешёвые зоны `.xyz`/`.online` — от ~1–3 $, `.com` ~9–11 $).
2. Зарегистрируйся на https://dash.cloudflare.com, нажми **Add a site**, введи свой домен. Cloudflare покажет **два своих nameserver-адреса** вида `xxx.ns.cloudflare.com` — скопируй их.
3. Вернись в Spaceship → открой домен → раздел **Nameservers** → выбери **Custom DNS** → вставь два адреса от Cloudflare → сохрани.
4. Подожди, пока Cloudflare подтвердит домен (от нескольких минут до пары часов) — статус станет **Active**, придёт письмо.

После этого Spaceship остаётся просто «владельцем» домена (продлеваешь раз в год), а маршрутизацией и HTTPS бесплатно рулит Cloudflare.

#### Шаг 1: установить и настроить туннель

Установи туннель-агент на сервер:

```bash
# скачать и поставить cloudflared
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb

# войти в свой аккаунт Cloudflare (откроется ссылка — открой её в браузере и подтверди)
cloudflared tunnel login

# создать туннель с именем
cloudflared tunnel create travelbot

# связать туннель с поддоменом (замени bot.ТВОЙ-ДОМЕН.com на свой)
cloudflared tunnel route dns travelbot bot.ТВОЙ-ДОМЕН.com
```

#### Шаг 2: создать файл конфигурации туннеля

```bash
mkdir -p ~/.cloudflared
nano ~/.cloudflared/config.yml
```

Вставь (замени `ТВОЙ-ДОМЕН` и `<ID-туннеля>` — ID показала команда `tunnel create`, также он есть в имени файла `~/.cloudflared/<ID>.json`):

```yaml
tunnel: <ID-туннеля>
credentials-file: /home/ТВОЙ-ПОЛЬЗОВАТЕЛЬ/.cloudflared/<ID-туннеля>.json

ingress:
  - hostname: bot.ТВОЙ-ДОМЕН.com
    service: http://localhost:8000
  - service: http_status:404
```

#### Шаг 3: запустить туннель как постоянную службу

(будет стартовать сам после перезагрузки)

```bash
sudo cloudflared service install
sudo systemctl start cloudflared
sudo systemctl enable cloudflared
```

#### Шаг 4: проверить снаружи

С телефона/другого компьютера открой в браузере:

```
https://bot.ТВОЙ-ДОМЕН.com/health
```

Видишь `{"status":"ok"}` — **твой публичный адрес готов**. Запиши его, он нужен в шаге 7.
Адрес webhook будет: `https://bot.ТВОЙ-ДОМЕН.com/webhook/instagram`

### Способ Б — DuckDNS + проброс портов 🟡

Подходит, если провайдер выдаёт тебе **«белый» (публичный) IP** и есть доступ к настройкам роутера. В проекте уже заложена поддержка DuckDNS (бесплатный динамический домен).

1. **Зарегистрируйся на https://www.duckdns.org** (через Google/GitHub), создай поддомен, например `mytravelbot` → получишь адрес `mytravelbot.duckdns.org` и **token**. Впиши их в `.env`:
   ```
   DUCKDNS_DOMAIN=mytravelbot
   DUCKDNS_TOKEN=твой-токен
   ```

2. **Узнай локальный IP сервера** (вида `192.168.x.x`):
   ```bash
   hostname -I
   ```

3. **Настрой проброс портов на роутере** (Port Forwarding). Зайди в админку роутера (обычно `192.168.0.1` или `192.168.1.1` в браузере) и пробрось:
   - внешний порт `80` → локальный IP сервера, порт `80`;
   - внешний порт `443` → локальный IP сервера, порт `443`.
   (Порты 80/443 — стандартные для HTTP/HTTPS; они нужны для автоматического получения сертификата.)

4. **Поставь Caddy** — он сам получит бесплатный HTTPS-сертификат и будет перенаправлять запросы в наш бот на порт 8000. Создай файл `Caddyfile` в папке проекта:
   ```bash
   nano Caddyfile
   ```
   Содержимое (замени домен):
   ```
   mytravelbot.duckdns.org {
       reverse_proxy localhost:8000
   }
   ```

5. Добавь Caddy в запуск рядом с ботом. Создай файл `docker-compose.override.yml`:
   ```bash
   nano docker-compose.override.yml
   ```
   Содержимое:
   ```yaml
   services:
     caddy:
       image: caddy:2
       container_name: caddy
       network_mode: host
       restart: unless-stopped
       volumes:
         - ./Caddyfile:/etc/caddy/Caddyfile:ro
         - caddy_data:/data
   volumes:
     caddy_data:
   ```

6. Перезапусти всё:
   ```bash
   docker compose up -d
   ```

7. Проверь снаружи (с телефона по мобильному интернету, не по домашнему Wi-Fi):
   ```
   https://mytravelbot.duckdns.org/health
   ```
   `{"status":"ok"}` — готово. Webhook: `https://mytravelbot.duckdns.org/webhook/instagram`

> Если снаружи не открывается: чаще всего провайдер не даёт белый IP (тогда используй **Способ А**) или порты 80/443 заблокированы/неправильно проброшены.

## 7. Подключаем webhook в Meta (Instagram)

Теперь скажем Meta, куда слать сообщения.

1. Зайди на https://developers.facebook.com → твоё приложение → раздел **Webhooks** (или Instagram → Configuration).
2. Нажми **Edit / Add Callback URL** и впиши:
   - **Callback URL:** твой публичный адрес + `/webhook/instagram`
     (например `https://bot.ТВОЙ-ДОМЕН.com/webhook/instagram`)
   - **Verify Token:** ровно та строка, что ты вписал в `.env` как `INSTAGRAM_VERIFY_TOKEN`.
3. Нажми **Verify and Save**. Meta дёрнет наш адрес для проверки — бот ответит автоматически. Если сохранилось без ошибки — связь установлена.
4. Подпишись на события сообщений (поля `messages`) для Instagram.

**Проверка, что сообщения реально доходят.** Напиши боту в Instagram и открой:

```
https://ТВОЙ-АДРЕС/webhook/instagram/last_seen
```

- `{"received_ever":true, ...}` — Meta достучалась, всё работает;
- `{"received_ever":false}` — сообщения не приходят (проверь, что приложение в Live Mode и аккаунт добавлен в тестировщиков/роли приложения).

## 8. Эксплуатация (повседневные команды)

Все команды — из папки `~/travel-agent-bot`.

**Посмотреть логи:**
```bash
docker compose logs -f          # в реальном времени (выход — Ctrl+C)
docker compose logs --tail 100  # последние 100 строк
```

**Перезапустить бот:**
```bash
docker compose restart
```

**Остановить / запустить:**
```bash
docker compose down    # остановить
docker compose up -d   # запустить снова
```

**Обновить бот после изменений в коде (например, после правок промпта):**
```bash
cd ~/travel-agent-bot
git pull origin master
docker compose up -d --build
```

**Сбросить сессии (если бот «запутался» в контексте):**
```bash
docker compose down
rm -f data/sessions.db
docker compose up -d
```

**Резервная копия данных.** Все данные (история, заявки, база FAQ) лежат в папке `data/` и файлах `.env` / `credentials.json`. Бэкап = просто скопировать их:
```bash
cp -r data ~/backup-$(date +%F)
cp .env credentials.json ~/backup-$(date +%F)/
```
Делай бэкап `data/` хотя бы раз в неделю (например, копируй на флешку/в облако).

**Автозапуск после перезагрузки.** Ничего настраивать не нужно: в `docker-compose.yml` стоит `restart: unless-stopped`, поэтому Docker сам поднимет бот после включения компьютера. Туннель `cloudflared` мы тоже добавили в автозапуск в шаге 6А.

## 9. Если что-то не работает (чеклист)

| Симптом | Что проверить |
|---|---|
| `curl http://localhost:8000/health` не отвечает | Бот не запустился. Смотри `docker compose logs --tail 100` — обычно ошибка в `.env` (пустой/неверный ключ). |
| Снаружи `https://адрес/health` не открывается | Способ А: жив ли туннель — `sudo systemctl status cloudflared`. Способ Б: белый IP и проброс портов 80/443. |
| Meta не сохраняет webhook | `INSTAGRAM_VERIFY_TOKEN` в `.env` ≠ Verify Token в Meta, либо адрес без `/webhook/instagram`. |
| Сообщения не приходят (`received_ever:false`) | Приложение Meta не в Live Mode, или Instagram-аккаунт не привязан/не в ролях приложения. |
| Бот молчит после ответа менеджера | Так и задумано: при ответе живого менеджера бот ставит паузу на `MANAGER_TAKEOVER_TTL_MINUTES` (по умолчанию 7 дней). Сбросить: `POST /api/admin/reset-takeover/{client_id}`. |
| Заявки не пишутся в Google Sheets | Нет/неверный `credentials.json`, либо таблица не «расшарена» на сервисный аккаунт из этого файла. |

## Если сервер на Windows

Можно и на Windows-компьютере:

1. Установи **Docker Desktop for Windows** (https://www.docker.com/products/docker-desktop) — он включает Docker Compose.
2. Установи **Git for Windows**, открой «Git Bash».
3. Дальше всё то же самое: `git clone ...`, настрой `.env` и `credentials.json`, `docker compose up -d --build`.
4. Публичный адрес — тот же **Способ А (Cloudflare Tunnel)**: у Cloudflare есть установщик `cloudflared` для Windows (.msi).
5. В настройках Docker Desktop включи автозапуск при старте Windows, а сам компьютер настрой так, чтобы не уходил в сон (Параметры → Питание → «Никогда»).

> Для постоянно работающего сервера Linux обычно надёжнее и экономнее по памяти, но Windows-вариант рабочий, если другого компьютера нет.
