# ИИ‑помощник турагентства — План реализации

## Архитектура
- **Язык:** Python 3.11+
- **Фреймворк AI:** LangGraph (нелинейный диалог)
- **LLM:** DeepSeek V4 (DeepSeek-R2) через API (с переключателем на V3 в .env)
- **Web‑сервер:** FastAPI
- **Канал (MVP):** Instagram Direct через Meta Graph API (Webhook)
- **База туров + заявки:** Google Sheets API
- **RAG/FAQ:** ChromaDB (локальная векторная БД)
- **Хранилище сессий:** SQLite
- **Уведомления менеджерам:** Telegram Bot
- **Деплой:** Docker → Timeweb Cloud VPS

## Функциональные требования

### 1. Приём заявок из Instagram Direct
- Webhook‑сервер на FastAPI, принимающий сообщения от Instagram Graph API
- Верификация webhook (hub.challenge)
- Обработка входящих текстовых сообщений
- Отправка ответов через Instagram Graph API (send message)

### 2. Классификация запроса
- Агент анализирует сообщение клиента и выделяет ключевые параметры:
  - Направление (страна / город)
  - Даты (или месяц / сезон)
  - Бюджет
  - Количество человек
  - Тип отдыха (пляж, экскурсии, горы и т.д.)
- Если параметров недостаточно — задаёт уточняющие вопросы (по одному, естественно)

### 3. База туров и выдача ссылок
- Агент имеет доступ к Google Sheets с полями: название тура, направление, тип, ключевые слова, даты, цена, ссылка на Google Docs
- По запросу клиента находит подходящие туры (фильтрация + fuzzy matching)
- Отправляет клиенту ссылки на Google Docs с полным описанием
- Если точного соответствия нет — предлагает ближайшие варианты и фиксирует пожелания

### 4. Сбор контактов и запись
- После отправки ссылок агент предлагает «Записаться на тур» или «Получить консультацию менеджера»
- При согласии последовательно собирает: имя → телефон → email → выбранный тур
- Данные сразу сохраняются в Google Sheets
- Клиенту отправляется подтверждение

### 5. Уведомление менеджера (эскалация)
Триггеры для передачи менеджеру:
- Клиент прямо просит поговорить с человеком
- Запрос слишком сложный / нестандартный
- Агент не нашёл подходящих туров
- Клиент хочет индивидуальный подбор
- Обнаружена фраза «срочно» / «жалоба»
- Клиент заполнил форму «записаться на консультацию»

Формат уведомления (Telegram):
- Контакты клиента (имя, телефон, email)
- Суть запроса
- История переписки (последние N сообщений)
- Тег «Нужен звонок»
- Ссылка на заявку в Google Sheets

### 6. Дополнительно
- Напоминания клиенту (за 24ч / 1ч до консультации)
- Ссылка на бронировочную форму / оплату (если возможно)
- База знаний FAQ (визы, страховка, погода, правила въезда) — RAG через ChromaDB

## Этапы реализации

### Этап 1: Настройка окружения (~2–3 дня)
- Инициализация Python‑проекта (FastAPI + uvicorn)
- requirements.txt (langgraph, langchain, google-api-python-client, chromadb, httpx)
- Dockerfile + docker-compose.yml
- .env с конфигами (DEEPSEEK_API_KEY, INSTAGRAM_ACCESS_TOKEN, GOOGLE_SHEETS_CREDENTIALS, TELEGRAM_BOT_TOKEN)
- config.py — загрузка переменных окружения

### Этап 2: Instagram API интеграция (~3–4 дня)
- Webhook‑сервер (FastAPI endpoint для Instagram Webhook)
- Верификация webhook (hub.challenge)
- Обработчик входящих сообщений (получение текста от клиента)
- Отправка ответов через Instagram Graph API (send message)
- Настройка ngrok для локальной разработки → позже смена на Timeweb домен

### Этап 3: AI‑движок LangGraph (~5–7 дней)
- Граф состояний диалога:
  - **GREETING** → приветствие, определение цели клиента
  - **CLASSIFY** → классификация: подбор тура / FAQ / жалоба / записаться
  - **CLARIFY** → извлечение параметров (направление, даты, бюджет, кол-во человек, тип отдыха)
  - **SEARCH_TOURS** → поиск подходящих туров в Google Sheets
  - **PRESENT_TOURS** → формирование ответа со ссылками на Google Docs
  - **BOOK** → предложение записаться, сбор контактов (имя, телефон, email)
  - **ESCALATE** → триггер передачи менеджеру
  - **FAQ** → поиск ответа в базе знаний через RAG
- Системные промпты на русском языке (дружелюбный экспертный тон, эмодзи)
- Управление контекстом диалога (память о предыдущих сообщениях)

### Этап 4: Google Sheets интеграция (~2–3 дня)
- Чтение базы туров (поля: название, направление, тип, даты, цена, ключевые слова, ссылка на Google Docs)
- Умный поиск: фильтрация по параметрам + fuzzy matching по ключевым словам
- Запись заявок в отдельный лист Google Sheets (имя, телефон, email, выбранный тур, дата, статус)
- История заявок для каждого клиента

### Этап 5: RAG — база знаний FAQ (~2–3 дня)
- ChromaDB: хранение эмбеддингов FAQ
- Загрузка FAQ из текстовых файлов / Google Docs
- Поиск релевантного ответа по вопросу клиента
- Темы: визы, страховка, погода, правила въезда, документы
- Кеширование частых запросов

### Этап 6: Сбор контактов и запись (~2–3 дня)
- После выдачи ссылок → предложение «Записаться на тур» / «Получить консультацию менеджера»
- Последовательный сбор: имя → телефон → email → подтверждение
- Валидация данных (формат телефона, email)
- Сохранение в Google Sheets + SQLite
- Отправка подтверждения клиенту

### Этап 7: Уведомления менеджерам (~2–3 дня)
- Telegram‑бот для уведомлений
- Триггеры эскалации (см. раздел 5)
- Формат уведомления: карточка с контактами, суть запроса, история переписки, тег «Нужен звонок»

### Этап 8: Доп. функции (напоминания, бронирование) (~2–3 дня)
- Напоминания клиенту (за 24ч / 1ч до консультации) — через Instagram + Telegram //не нужно
- Ссылка на бронировочную форму / оплату (если есть)
- Статусы заявок (новая → в обработке → подтверждена → оплачена)

### Этап 9: Тестирование и деплой (~3–5 дней)
- Unit‑тесты для AI‑движка
- Integration‑тесты для каналов
- Ручное тестирование сценариев диалога
- Docker‑деплой на Timeweb Cloud
- SSL сертификат, домен
- Мониторинг (логи, алерты)
- Документация (SETUP.md)

## Примечания для разработчика
- Код спроектирован с абстрактными интерфейсами каналов — добавление Telegram, WhatsApp, VK требует только нового адаптера в `src/channels/`
- LLM переключается через `.env` переменную `DEEPSEEK_MODEL`
- Google Sheets достаточно для старта — при росте проекта предусмотрена лёгкая миграция в CRM (AmoCRM / Bitrix24)
- Все ответы агент генерирует на русском языке, в дружелюбном экспертном тоне, с эмодзи
- Сценарий диалога — гибкий, нелинейный (LangGraph state machine)
- Регион: РБ (Республика Беларусь) — учтено при выборе хостинга и платежных методов
