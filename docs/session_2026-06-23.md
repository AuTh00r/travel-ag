# Сессия: 23.06.2026 — Улучшение ответов бота + багфикс дубляжа

## Что сделано

### Шаг 1–2. Нода `handle_tour_selection` + редизайн графа

**Файлы:**
- `src/ai/engine.py` — новая нода `handle_tour_selection`, conditional edges
- `src/ai/nodes.py` — реализация ноды
- `src/ai/prompts.py` — новый промпт `SELECT_TOUR_PROMPT`

**Суть:**
Раньше после `present_tours` граф шёл в `END`. Выбор тура пользователем не обрабатывался (selected_tour всегда None). Следующее сообщение уходило в полный цикл `START → classify`, теряя контекст.

**Теперь:**
```
present_tours → handle_tour_selection → route_after_selection()
  ├─ "select"   → book (устанавливает selected_tour, ASK_NAME)
  ├─ "retry"    → clarify (сброс параметров, новый поиск)
  ├─ "escalate" → escalate (Telegram уведомление)
  └─ "ask_again" → END (переспрашиваем, ждём ответ)
```

Нода использует LLM (JSON mode) для определения намерения пользователя после показа туров.

### Шаг 3. Контекст диалога во все промпты

**Файлы:**
- `src/ai/prompts.py` — новая функция `build_context()`, плейсхолдер `{context}` во всех 6 промптах
- `src/ai/classifier.py` — контекст передаётся в `CLASSIFICATION_PROMPT`
- `src/ai/nodes.py` — контекст в `EXTRACT_PROMPT`, `CLARIFY_PROMPT`, `PRESENT_TOURS_PROMPT`, `SELECT_TOUR_PROMPT`
- `src/ai/faq.py` — контекст в `FAQ_PROMPT`

**Суть:**
Раньше все LLM-вызовы получали только текущее сообщение и известные параметры, без истории диалога. LLM не знал, что БОТ уже спросил, и пользователь ответил. Теперь каждому вызову передаются последние 5 сообщений как контекст.

### Шаг 4. Структурированный `present_tours`

**Файлы:** `src/ai/nodes.py`

**Суть:**
Раньше туры форматировались как `"1. Название — Направление — Цена"`, без ссылок на Google Docs. Теперь каждый тур включает название, направление, тип, даты, цену, длительность и ссылку в структурированном виде.

### Шаг 5. LLM-based booking (частично)

**Файлы:** `src/ai/nodes.py`

**Суть:**
Раньше `book/AWAIT_NAME` брал `last_message.strip()` как имя. Пользователь пишет «меня зовут Иван» → имя = «меня зовут Иван». Теперь LLM извлекает чистое имя через `_EXTRACT_NAME_PROMPT`.

### Багфикс: дубляж сообщений при escalation

**Файлы:** `src/ai/nodes.py`

**Проблема:**
Когда туров не найдено, `present_tours` возвращал своё сообщение («не нашёл туров») + выставлял `needs_escalation=True`. Граф шёл в `escalate`, который отправлял второе сообщение («передал менеджеру»). Instagram получал 2 сообщения подряд, клиент показывал ошибку «не удалось загрузить предыдущие сообщения».

**Фикс:**
`present_tours` при 0 турах больше не добавляет AIMessage. Только `escalate` отвечает пользователю. Сообщение адаптируется под причину escalation.

## Известные проблемы

1. **Мало туров в Google Sheets** — всего 1 тур (Анталья All-Inclusive, 1200$). На запрос «Кушадасы» находится 0 результатов, бот эскалирует.
2. **Instagram send token** — возможно короткоживущий, при ошибках отправки сообщения клиент видит «не удалось загрузить предыдущие сообщения». В логах пока ошибок нет.
3. **BOOK_PROMPT не используется** — определён в `prompts.py` (строка 160), но не вызывается из `book()`. Сейчас используется хардкод + LLM для извлечения имени.
4. **`greet()` не вызывается** — из-за логики в `route_from_start()` первое сообщение пользователя (которое уже есть в `messages`) идёт сразу в `classify`, минуя greeting.
